# Contributing to the Sindri extension registry

This repository is the first-party extension registry for Sindri IDE. It is also the **reference layout for third-party registries** — host your own git repository with the same structure and Sindri's marketplace will discover and load it.

## Repository layout

```text
sindri-extensions/
├── index.json                ← registry index (auto-generated by CI — do not edit)
├── manifest.schema.json      ← JSON Schema; IDEs validate manifest.json against this
├── CONTRIBUTING.md           ← this file
│
├── aurora-theme/             ← one folder per extension
│   ├── manifest.json         ← required — extension identity and contributions
│   ├── README.md             ← required — shown as the marketplace detail page
│   └── aurora-dark.json      ← ThemeDef JSON referenced in manifest.json
│
└── neon-icons/
    ├── manifest.json
    ├── README.md
    └── icons.json            ← IconThemeDef JSON referenced in manifest.json
```

`index.json` at the repository root is regenerated automatically by CI on every push. **Do not edit it by hand** — your changes will be overwritten.

## Adding an extension

1. Create a new folder with a short, lowercase, hyphen-separated name matching the `name` part of your `id` (e.g. `my-rust-theme`).
2. Add a `manifest.json` with `"$schema": "../manifest.schema.json"` at the top — this gives you IDE validation and autocompletion.
3. Add a `README.md` — this is the detail page shown in the marketplace when a user clicks on your extension.
4. Add the asset files referenced by `manifest.json` (`path` fields in `contributes.themes`, `contributes.iconThemes`, etc.).
5. Open a pull request. CI validates the manifest against the schema and regenerates `index.json`.

## manifest.json field reference

Add `"$schema": "../manifest.schema.json"` as the first field and your IDE will validate the file automatically.

### `id`

**Required.** Format: `publisher.name` — lowercase letters, numbers, and hyphens only. Must be globally unique across the registry.

```json
"id": "yourname.my-theme"
```

The `publisher` part must match the top-level `publisher` field.

---

### `name`, `version`, `publisher`, `description`

**All required.**

| Field | Type | Description |
| --- | --- | --- |
| `name` | string | Human-readable display name shown in the marketplace card |
| `version` | string | Semantic version in `major.minor.patch` format. Must match a git tag in the repository |
| `publisher` | string | Publisher identifier — must match the prefix of `id` |
| `description` | string | One-sentence description shown in the marketplace card |

```json
"name": "My Theme",
"version": "1.0.0",
"publisher": "yourname",
"description": "A warm dark theme for Sindri IDE."
```

---

### `categories`

**Required.** Array of marketplace browsing facets. An extension may belong to multiple categories.

| Value | Description | Installable now? |
| --- | --- | --- |
| `"Color Theme"` | UI colour palette — applies CSS custom properties (colours, glows) from a ThemeDef JSON file | ✅ Yes |
| `"File Icon Theme"` | File tree icon set — an IconThemeDef JSON with inline SVGs mapped to file extensions and names | ✅ Yes |
| `"UI Icon Theme"` | Activity bar and toolbar glyphs — a UiIconPackDef JSON mapping tool-window IDs to SVG strings | ✅ Yes |
| `"Language Support"` | Programming language integration — Tree-sitter grammar, LSP language server, DAP debugger adapter | 🚧 Requires extension host |
| `"Language Pack"` | Human-language UI localisation — translations for all Sindri UI strings | 🚧 Requires extension host |
| `"Test & Task Adapter"` | Test runner or build task adapter | 🚧 Requires extension host |
| `"UI Extension"` | General workbench UI additions — custom panels, editor decorations, status bar items | 🚧 Requires extension host |

Data-only categories (`Color Theme`, `File Icon Theme`, `UI Icon Theme`) can be installed today. Code-bearing extensions require the QuickJS extension host, which is in development.

```json
"categories": ["Color Theme"]
```

```json
"categories": ["Color Theme", "File Icon Theme"]
```

---

### `permissions`

**Required.** List of `sindri.*` API namespaces the extension is allowed to call. Data-only extensions must pass an empty array.

| Permission | What it grants |
| --- | --- |
| *(empty array)* | Data-only — no host APIs called |
| `"sindri.workspace"` | Read the open workspace file tree and metadata |
| `"sindri.fs"` | Read and write files on disk (beyond workspace metadata) |
| `"sindri.terminal"` | Open terminals and run shell commands |
| `"sindri.lsp"` | Register and communicate with Language Server Protocol servers |
| `"sindri.dap"` | Register and communicate with Debug Adapter Protocol servers |
| `"sindri.ui"` | Register custom panels, status bar items, and editor decorations |
| `"sindri.settings"` | Read and write Sindri settings (own namespace only) |
| `"sindri.network"` | Make outbound network requests |

Sindri shows each requested permission to the user before installation. Request only what your extension genuinely needs — unnecessary permissions reduce user trust.

```json
"permissions": []
```

```json
"permissions": ["sindri.workspace", "sindri.lsp"]
```

---

### `engines`

**Required.** Declares the minimum Sindri version this extension is compatible with. Sindri will refuse to install an extension whose `engines.sindri` range excludes the current version.

| Value | Meaning |
| --- | --- |
| `">=0.1.0"` | Compatible with Sindri 0.1.0 and all later versions — recommended for new extensions |
| `">=1.0.0"` | Requires Sindri 1.0.0 or later |
| `"^1.2.0"` | Compatible with 1.2.x and 1.3.x etc., but not 2.0 |
| `">=0.2.0 <1.0.0"` | Compatible with 0.2.x up to (not including) 1.0 |

```json
"engines": { "sindri": ">=0.1.0" }
```

---

### `contributes`

**Required** (but all sub-fields are optional — include only what applies).

| Sub-field | Extension type | Description |
| --- | --- | --- |
| `contributes.themes[]` | Color Theme | Array of ThemeDef JSON file references |
| `contributes.iconThemes[]` | File Icon Theme, UI Icon Theme | Array of IconThemeDef JSON file references |
| `contributes.lsp[]` | Language Support | Language Server Protocol adapter declarations |
| `contributes.dap[]` | Language Support | Debug Adapter Protocol adapter declarations |
| `contributes.grammars[]` | Language Support | Tree-sitter grammar file references |
| `contributes.taskAdapters[]` | Test & Task Adapter | Task runner adapter declarations |
| `contributes.configuration` | Any | Settings schema contributed to the Sindri Settings panel |

See the [extension type guides](#extension-type-guides) below for the structure of each sub-field.

---

### main

Path to the compiled extension entry-point bundle, relative to the extension folder. **Omit this field entirely for data-only extensions** (themes, icons). Its presence signals that an extension host is required.

```json
"main": "dist/extension.js"
```

## Extension type guides

### Color Theme

A Color Theme extension ships one or more ThemeDef JSON files and declares them under `contributes.themes`.

**manifest.json**

```json
{
  "$schema": "../manifest.schema.json",
  "id": "yourname.my-theme",
  "name": "My Theme",
  "version": "1.0.0",
  "publisher": "yourname",
  "description": "A warm dark theme for Sindri IDE.",
  "categories": ["Color Theme"],
  "permissions": [],
  "engines": { "sindri": ">=0.1.0" },
  "contributes": {
    "themes": [
      { "id": "my-theme-dark",  "name": "My Theme Dark",  "kind": "dark",  "path": "dark.json" },
      { "id": "my-theme-light", "name": "My Theme Light", "kind": "light", "path": "light.json" }
    ]
  }
}
```

`contributes.themes[]` fields:

| Field | Type | Description |
| --- | --- | --- |
| `id` | string | Unique ID for this specific theme variant |
| `name` | string | Display name in the Appearance dropdown |
| `kind` | `"dark"` \| `"light"` | Controls `data-theme-kind` on `<html>` — affects system scrollbars and OS chrome |
| `path` | string | Path to the ThemeDef JSON file, relative to the extension folder |

See [ThemeDef format reference](#themedef-format-reference) for the full JSON structure of the theme file.

---

### File Icon Theme

A File Icon Theme extension ships one or more IconThemeDef JSON files for the file explorer.

**manifest.json**

```json
{
  "$schema": "../manifest.schema.json",
  "id": "yourname.my-icons",
  "name": "My Icons",
  "version": "1.0.0",
  "publisher": "yourname",
  "description": "A minimal file icon theme.",
  "categories": ["File Icon Theme"],
  "permissions": [],
  "engines": { "sindri": ">=0.1.0" },
  "contributes": {
    "iconThemes": [
      { "id": "my-icons-color", "name": "My Icons",      "kind": "color", "path": "icons.json" },
      { "id": "my-icons-mono",  "name": "My Icons Mono", "kind": "mono",  "path": "icons.json" }
    ]
  }
}
```

`contributes.iconThemes[]` fields:

| Field | Type | Description |
| --- | --- | --- |
| `id` | string | Unique ID for this icon theme variant |
| `name` | string | Display name in the Appearance dropdown |
| `kind` | `"color"` \| `"mono"` | `"color"` uses full-colour SVGs; `"mono"` uses `currentColor` (inherits the theme accent colour) and prefers `monoSvg` if present |
| `path` | string | Path to the IconThemeDef JSON file |

A single JSON file can serve both a color and a mono variant — provide `monoSvg` alongside `svg` in each icon entry. See [IconThemeDef format reference](#iconthemedef-format-reference).

---

### UI Icon Theme

A UI Icon Theme provides the glyphs used in the activity bar and toolbar (tool-window icons).

**manifest.json**

```json
{
  "$schema": "../manifest.schema.json",
  "id": "yourname.my-ui-icons",
  "name": "My UI Icons",
  "version": "1.0.0",
  "publisher": "yourname",
  "description": "A compact icon pack for the Sindri activity bar.",
  "categories": ["UI Icon Theme"],
  "permissions": [],
  "engines": { "sindri": ">=0.1.0" },
  "contributes": {
    "iconThemes": [
      { "id": "my-ui-icons", "name": "My UI Icons", "kind": "mono", "path": "ui-icons.json" }
    ]
  }
}
```

The JSON format is the same `IconThemeDef` as for File Icon Themes. Tool-window IDs (keys in the `icons` map) correspond to the panel identifiers registered in the workbench (`explorer`, `search`, `git`, `debug`, `extensions`, `terminal`, `problems`, `output`, `settings`, etc.).

### Language Support

A Language Support extension bundles a Tree-sitter grammar and/or declares LSP/DAP servers. **Requires the extension host** — declare this in `categories` and list the `sindri.lsp` / `sindri.dap` permissions.

**manifest.json**

```json
{
  "$schema": "../manifest.schema.json",
  "id": "yourname.rust-support",
  "name": "Rust Support",
  "version": "1.0.0",
  "publisher": "yourname",
  "description": "Rust language support — syntax, LSP (rust-analyzer), and DAP (lldb).",
  "categories": ["Language Support"],
  "permissions": ["sindri.workspace", "sindri.lsp", "sindri.dap"],
  "engines": { "sindri": ">=0.1.0" },
  "contributes": {
    "grammars": [
      { "languageId": "rust", "path": "grammars/tree-sitter-rust.wasm" }
    ],
    "lsp": [
      {
        "id": "rust-analyzer",
        "languageId": "rust",
        "launch": { "command": "rust-analyzer" }
      }
    ],
    "dap": [
      {
        "id": "lldb",
        "languageId": "rust",
        "launch": { "command": "lldb-vscode", "args": [] }
      }
    ]
  },
  "main": "dist/extension.js"
}
```

`contributes.lsp[]` and `contributes.dap[]` fields:

| Field | Type | Description |
| --- | --- | --- |
| `id` | string | Unique adapter ID within this extension |
| `languageId` | string | Language identifier (e.g. `"rust"`, `"python"`, `"typescript"`) |
| `launch.command` | string | Executable name or path to the server binary |
| `launch.args` | string[] | Optional command-line arguments |

`contributes.grammars[]` fields:

| Field | Type | Description |
| --- | --- | --- |
| `languageId` | string | Language identifier |
| `path` | string | Path to the compiled `.wasm` grammar file |

### Language Pack

A Language Pack provides translated UI strings for a human language.

**manifest.json**

```json
{
  "$schema": "../manifest.schema.json",
  "id": "yourname.lang-fr",
  "name": "Français",
  "version": "1.0.0",
  "publisher": "yourname",
  "description": "French language pack for Sindri IDE.",
  "categories": ["Language Pack"],
  "permissions": [],
  "engines": { "sindri": ">=0.1.0" },
  "main": "dist/extension.js"
}
```

The extension's `main` entry uses the `sindri.ui.registerLocale(locale, translations)` API (extension host required). The translation file format will be documented when the localisation API stabilises.

## ThemeDef format reference

A ThemeDef JSON file defines a complete colour theme. All fields are required.

```json
{
  "id": "my-theme-dark",     // must match contributes.themes[].id in manifest.json
  "name": "My Theme Dark",   // display name
  "kind": "dark",            // "dark" | "light"

  // UI chrome — CSS custom properties applied to :root
  "ui": {
    "--bg":            "#0D0D14",   // main editor / workbench background
    "--bg-panel":      "#111118",   // panel / sidebar background
    "--bg-chrome":     "#0A0A10",   // header bars, nav chrome
    "--bg-hover":      "rgba(255,255,255,0.05)",  // hover highlight (use rgba)
    "--border":        "#1E1E28",   // primary border colour
    "--border-subtle": "#161620",   // de-emphasised borders
    "--text":          "#C8C8D8",   // primary text
    "--text-dim":      "#60607A",   // secondary / muted text
    "--accent":        "#7C6BFF",   // brand accent — buttons, active items, cursor
    "--accent-tint":   "rgba(124,107,255,0.14)",  // accent at low opacity for backgrounds
    "--danger":        "#FF5560"    // error / destructive colour
  },

  // Glow effects (used on focused elements and the settings dialog shadow)
  "glow": {
    "--glow-color":         "124,107,255",   // RGB components of accent for rgba()
    "--glow-accent":        "0 0 10px rgba(124,107,255,.5)",
    "--glow-accent-strong": "0 0 18px rgba(124,107,255,.65),0 0 36px rgba(124,107,255,.28)"
  },

  // CodeMirror editor surface — self-contained; must NOT reference the CSS vars above
  "editor": {
    "bg":              "#0A0A12",   // editor background
    "fg":              "#C8C8D8",   // default text colour
    "caret":           "#7C6BFF",   // text cursor
    "selection":       "#1E1C40",   // text selection highlight
    "activeLine":      "#0E0E18",   // current line highlight
    "gutter.bg":       "#0A0A12",   // line number gutter background
    "gutter.fg":       "#303040",   // line number text
    "gutter.activeFg": "#6060A0",   // active line number text
    "matchingBracket": "#1E1C44"    // matching bracket highlight
  },

  // Syntax token colours
  "syntax": {
    "keyword":        { "color": "#A890FF" },
    "controlKeyword": { "color": "#A890FF" },
    "string":         { "color": "#60D0A0" },
    "number":         { "color": "#70B0FF" },
    "bool":           { "color": "#70B0FF" },
    "comment":        { "color": "#404060", "fontStyle": "italic" },
    "function":       { "color": "#C0A8FF" },
    "type":           { "color": "#B0C8FF" },
    "variable":       { "color": "#C8C8D8" },
    "property":       { "color": "#9090B8" },
    "operator":       { "color": "#7C6BFF" },
    "punctuation":    { "color": "#60607A" },
    "tag":            { "color": "#A890FF" },
    "attribute":      { "color": "#60D0A0" },
    "heading":        { "color": "#C0A8FF" },
    "link":           { "color": "#7C6BFF" },
    "regexp":         { "color": "#60D0A0" },
    "escape":         { "color": "#A890FF" }
  }
}
```

`syntax` token style objects support:

| Property | Type | Description |
| --- | --- | --- |
| `color` | string (hex/rgb) | Token colour — required |
| `fontStyle` | `"italic"` | Apply italic — optional |
| `fontWeight` | string | e.g. `"bold"` — optional |

## IconThemeDef format reference

An IconThemeDef JSON file maps file names, extensions, and language IDs to icon entries.

```json
{
  "id": "my-icons-color",    // must match contributes.iconThemes[].id
  "name": "My Icons",
  "kind": "color",           // "color" | "mono"

  // Map specific file names to icon IDs (highest priority)
  "fileNames": {
    ".gitignore":   "git",
    "package.json": "json",
    "Cargo.toml":   "toml",
    "Makefile":     "sh"
  },

  // Map file extensions to icon IDs (second priority)
  "fileExtensions": {
    "ts":   "ts",
    "tsx":  "tsx",
    "js":   "js",
    "json": "json",
    "rs":   "rs",
    "md":   "md"
  },

  // Map language IDs to icon IDs (third priority — future use)
  "languageIds": {
    "typescript": "ts",
    "rust":       "rs"
  },

  // Map folder names to icon IDs
  "folderNames": {
    "src":    "folder-src",
    "dist":   "folder-dist",
    ".git":   "folder-git"
  },

  // Fallback icons when no specific mapping matches
  "defaults": {
    "file":       "file",
    "folder":     "folder",
    "folderOpen": "folder-open"
  },

  // Icon definitions — inline SVG strings
  // Each key is an icon ID referenced above in fileExtensions, fileNames, etc.
  "icons": {
    "file": {
      "svg":     "<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 16 16'>...</svg>",
      "monoSvg": "<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 16 16'>...</svg>"
    },
    "ts": {
      "svg":     "<svg ...>...</svg>"
    }
  }
}
```

Resolution order (first match wins):

1. `fileNames[fileName]` — exact file name match
2. `fileExtensions[ext]` — file extension match (without the leading `.`)
3. `languageIds[langId]` — language ID match
4. `defaults.file` — fallback for files, `defaults.folder` / `defaults.folderOpen` for directories

`icons` entry fields:

| Field | Type | Description |
| --- | --- | --- |
| `svg` | string | Full-colour inline SVG string — used for `"color"` kind and as fallback for `"mono"` |
| `monoSvg` | string (optional) | Monochrome SVG using `currentColor` — preferred for `"mono"` kind |

SVGs should use `viewBox='0 0 16 16'` and be self-contained (no external references).

## index.json format

`index.json` is an array of entries. Each entry wraps a `manifest.json` with registry-level metadata added by CI:

```json
[
  {
    "manifest":       { /* the full contents of manifest.json */ },
    "tags":           ["v1.0.0", "v1.1.0"],   // git tags available in the repo
    "readmeContent":  "# My Extension\n...",  // README.md contents, inlined by CI
    "folderPath":     "my-extension"          // folder name within the repo
  }
]
```

You do not write `index.json` directly. CI reads every `manifest.json` in the repository, fetches the corresponding `README.md`, and regenerates `index.json` on each push.
