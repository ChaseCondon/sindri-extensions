# Contributing extensions to the Sindri registry

This repository is the first-party extension registry for [Sindri IDE](https://github.com/sindri-ide/sindri-ide). It is also the reference implementation for third-party registries — host your own git repo with the same layout and Sindri's marketplace will load it.

---

## Repository layout

```
sindri-extensions/
├── index.json            ← registry index (auto-generated by CI; do not edit by hand)
├── manifest.schema.json  ← JSON Schema for manifest.json files
├── CONTRIBUTING.md       ← this file
│
├── my-extension/
│   ├── manifest.json     ← required
│   ├── README.md         ← required (shown in the marketplace detail view)
│   └── <assets>          ← theme JSON, icon JSON, grammar files, etc.
│
└── another-extension/
    └── ...
```

`index.json` is regenerated from all `manifest.json` files by the CI pipeline on each push. **Do not edit `index.json` directly** — your changes will be overwritten.

---

## Adding an extension

1. Create a new folder with a short, lowercase, hyphen-separated name (e.g. `my-rust-theme`).
2. Add a `manifest.json` — see [manifest.schema.json](manifest.schema.json) for the full schema and field descriptions.
3. Add a `README.md` — this is the storefront detail page shown in the marketplace.
4. Add any asset files referenced by `manifest.json` (theme JSON, icon JSON, etc.).
5. Open a pull request. CI validates the manifest against the schema and regenerates `index.json`.

---

## manifest.json reference

See [manifest.schema.json](manifest.schema.json) for the machine-readable schema (IDEs will validate your file automatically if you add `"$schema": "../manifest.schema.json"` at the top).

### Minimal example — colour theme

```json
{
  "$schema": "../manifest.schema.json",
  "id": "yourname.my-theme",
  "name": "My Theme",
  "version": "1.0.0",
  "publisher": "yourname",
  "description": "A cosy dark theme for Sindri IDE.",
  "categories": ["Color Theme"],
  "permissions": [],
  "engines": { "sindri": ">=0.1.0" },
  "contributes": {
    "themes": [
      { "id": "my-theme-dark", "name": "My Theme Dark", "kind": "dark", "path": "dark.json" }
    ]
  }
}
```

### Minimal example — file icon theme

```json
{
  "$schema": "../manifest.schema.json",
  "id": "yourname.my-icons",
  "name": "My Icons",
  "version": "1.0.0",
  "publisher": "yourname",
  "description": "A clean file icon theme.",
  "categories": ["File Icon Theme"],
  "permissions": [],
  "engines": { "sindri": ">=0.1.0" },
  "contributes": {
    "iconThemes": [
      { "id": "my-icons-color", "name": "My Icons", "kind": "color", "path": "icons.json" }
    ]
  }
}
```

## Field reference

### `id`

Format: `publisher.name` — lowercase letters, numbers, and hyphens only. Must be globally unique across the registry.

```
"id": "sindri.aurora-theme"
       ──────  ────────────
       publisher  name
```

### `categories`

| Value | Description | Extension host required? |
|---|---|---|
| `"Color Theme"` | UI palette — applies CSS custom properties from a ThemeDef JSON | No |
| `"File Icon Theme"` | File tree icons — an IconThemeDef JSON with inline SVGs | No |
| `"UI Icon Theme"` | Activity bar / toolbar glyphs — a UiIconPackDef JSON | No |
| `"Language Support"` | Tree-sitter grammar + LSP / DAP adapters via Sindri Adapter Protocol | Yes (QuickJS host) |
| `"Language Pack"` | Human-language UI localisation | Yes (QuickJS host) |
| `"Test & Task Adapter"` | Test runner or task adapter | Yes (QuickJS host) |
| `"UI Extension"` | General workbench UI additions | Yes (QuickJS host) |

An extension may declare multiple categories (e.g. an extension that ships both a colour theme and an icon theme).

Data-only categories (`Color Theme`, `File Icon Theme`, `UI Icon Theme`) can be installed today without the extension host. Code-bearing extensions require the QuickJS host — currently in development.

### `permissions`

A list of `sindri.*` API namespaces the extension is allowed to call (ADR-0015 §6). Data-only extensions must pass an empty array. The IDE will refuse to install an extension that requests permissions it cannot grant.

```json
"permissions": []                              // data-only
"permissions": ["sindri.workspace"]           // read workspace file tree
"permissions": ["sindri.workspace", "sindri.terminal"]  // workspace + terminal
```

### `engines.sindri`

A semver range describing the minimum Sindri version required. Use `">=0.1.0"` to indicate compatibility with all 0.x releases. Sindri will refuse to install an extension that declares an incompatible engine range.

| Value | Meaning |
|---|---|
| `">=0.1.0"` | Any 0.1.0 or later (recommended for new extensions) |
| `">=1.0.0"` | Requires Sindri 1.0.0 or later |
| `"^1.2.0"` | Compatible with 1.2.x and 1.3.x etc., but not 2.0 |

### `contributes.themes[].kind`

| Value | Effect |
|---|---|
| `"dark"` | Sets `data-theme-kind="dark"` on `<html>` — dark scrollbars, dark system UI |
| `"light"` | Sets `data-theme-kind="light"` — light scrollbars, light system UI |

### `contributes.iconThemes[].kind`

| Value | Effect |
|---|---|
| `"color"` | Full-colour SVG icons; `svg` field is used |
| `"mono"` | Monochrome icons that inherit `currentColor`; `monoSvg` field is preferred, falls back to `svg` |

### `main` (code-bearing extensions only)

Path to the compiled extension entry point, relative to the extension folder. Omit this field entirely for data-only extensions. Its presence signals to the IDE that an extension host is required.

## Theme file format (ThemeDef)

See the [Aurora Dark example](aurora-theme/aurora-dark.json) and the authoritative TypeScript type in [`sindri-ide/src/theme/tokens.ts`](https://github.com/sindri-ide/sindri-ide/blob/main/src/theme/tokens.ts).

## Icon theme file format (IconThemeDef)

See the [Neon Icons example](neon-icons/icons.json) and the authoritative TypeScript type in [`sindri-ide/src/theme/tokens.ts`](https://github.com/sindri-ide/sindri-ide/blob/main/src/theme/tokens.ts).

Icons are inline SVG strings. Both `color` and `mono` (via `monoSvg`) variants can be provided in the same `icons` map.

---

## index.json format

`index.json` is an array of `RegistryIndexEntry` objects (see [`sindri-ide/src/extensions/manifest.ts`](https://github.com/sindri-ide/sindri-ide/blob/main/src/extensions/manifest.ts) for the TypeScript type). The CI pipeline populates `readmeContent` from each extension's `README.md` automatically — you do not need to add it to your manifest.
